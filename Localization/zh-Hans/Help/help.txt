&LOGO\\

!起源及目标\
C-evo is a non-commerical game project based on the famous "Sid Meier's
Civilization" by Microprose. The ambition of C-evo is not, to compete with
commercial games for the most exciting presentation. Instead, it aims at being a
true, ageless strategy game that entertains by one thing above all:
challenge. So the project focus is design of rules and AI. For more
information, visit the project homepage:\
::E2 c-evo.org\\

!所有权许可\
C-evo is a compilation of work by a number of authors. You are allowed
to play the game and to distribute it without explicit permission. The compilation
itself and all of its components (except for the graphics and the sounds) are in the Public
Domain. Means you can do with it whatever you want. This also includes the binary
code and the sources, which are available from the project homepage. As for
the graphics, you should ask the author of an item before using it for another
purpose than to play.\
::C 查看制作人员名单\\

!AI模块\
The game has an open AI interface, which means the player can replace the standard AI
contained in the package by other AI algorithms, either for all or for single
nations. C-evo allows playing with up to 8 different AIs in the same
game. The documentation of the AI's DLL-interface is available from the project
homepage. There is also an AI development kit, you can find it in the folder where
you installed the game. Maybe competition will lead us to the first real good AI
in modern strategy gaming.\
::E1 AI模块开发指南\\

!错误报告\

To support the progress of this project, please tell me about problems you have
encountered. But first you should\
-Look to the list of known bugs on the project homepage. Don't
report bugs which are already mentioned there.\
-Look to see if there is already a newer program version than yours. This might
have already fixed your problem, particularly if it's a quite conspicuous one.\
-Read the corresponding manual part. Maybe your bug isn't one. (This game is
not a Civilization II clone!)\
-Be sure it's not a problem with the installation or with some third party
expansion such as alternative AI. If you're not sure, uninstall the game with
the Windows Application Wizard, reinstall it with no expansion and then test
again. In the event of problems with an expansion, please contact its author.\\

Before reporting a problem, please try to find a way to reproduce it. A report
of a reproducable bug is much more valuable. To reproduce the bug it may well
be enough to load the book in the turn before the problem occured and continue
in the same way as before. This is even possible in case of a crash, because
there's always an autosave from the last turn, it's the book with the name
starting on "~".\\

Then, send me a detailed destription of the problem. Use the web form at\
::E3 c-evo.org/_sg/contact\
If your book is
necessary to reproduce the problem, please attach it. You can find the file in
the subfolder "Saved" of C-evo.

!致文明系列的老玩家\

就地形和部队图片的相似程度而言, C-evo(文明进化史)
很容易被看作是文明II代经典的复制版. 但深入游戏后, 
你就会发现两者有许多不同. 其差异程度甚至可与
文明I和II代之间的差异相提并论. 查看具体差异, 
请点击下列标题:\
:MOVEMENT 部队移动\
::T 地形种类\
::B202 奇迹影响\
::G 政体形式\
::A 科技演化\
:DIPLOMACY 对外交涉\
:SHIP 殖民飞船\\

除了上述差异之外, 还有以下要点提请各位玩家留意:\

-C-evo不靠随机运算作为卖点. 每一项计划的结果都可
预期, 除了世界地图的生成和其他势力的举动之外.\
-电脑计算所依据的规则与真人玩家完全相同. 电脑玩家
并无额外的优势, 在获得情报方面也一样.\
-城市结构与文明系列略有出入. 在城市所在的地块, 有1个
市民负责劳动生产, 但不会从事商业贸易; 此外, 市民结构
也有所简化, 如去除了艺人和收税人等等.\
-城市最小规模为2级, 生产一支拓荒者, 须耗费2个市民.\
-城市粮食储量上限并非由城市规模决定, 因此大城市的
增长速度, 往往远高于较小的城市.\
-无需临近海洋或河流, 即可进行灌溉.\
-从其他势力获得的先进科技, 无法直接利用, 必须先针对
该科技进行剖析性研究, 研究速度将是独自研究的两倍, 
且不必研究先期所需科技.\
-某些高级作战部队无法直接建设, 必须先进行研发, 
研发成功后才能建设.\
-某些地形对地面部队会造成伤害, 详见"地形种类"中的"恶土".\

!概论\

国家的组成单位是部队和城市. 部队的职责是探险和攻守, 
而城市的职能是人口增长, 生产建设和科学研究.\\

世界地图由四边形地块组成. 每个地块都有其地形属性, 
根据不同的地形种类, 地块的以下三种属性会有所不同: 
资源产量, 防御加成, 和移动消耗. 同一地块上最多能建
一座城市, 但可容纳的同国部队数量并无限制. 要获得
特定地块的帮助信息, 请在点选地块时按住Shift键.\

所有部队行动完成后,回合将自动结束, 如果您看到"等待
回合结束"的提示,请点击屏幕右下角的结束回合按钮.\\

#DIPLOMACY
$外交\

!概论\

两个素未谋面的国家要建立外交关系, 必须等到其中一国
的地面部队发现另一国的地面部队或城市之后方可进行. 
在国家情报画面, 可查看已知其他势力的相关信息. 
在国家情报画面中, 显示了一个国家的政体形式, 现有资金, 
外交立场, 以及人口, 领土, 科技, 探索等概况. \\

!交涉\
当两国发生接触之后, 可开展外交交涉, 在交涉中可进行
交换贸易或签署协约. 整个交涉过程主要由制定提议, 
接受提议, 以及取消条约三部分组成.\\

当交涉双方都停止提议的时候, 交涉就将结束. 如果一方
选择终止谈判, 交涉也会立即终止.\\

!提议\
外交交涉主要的组成部分是各种提议. 提议的内容中包含了
交涉一方向另一方提出的要求, 以及愿意为之付出的条件. 
可能包含的内容有:\
-一项条约\
-国家情报\
-军事情报\
-一笔现金\
-科技成果\
-部队研发\
-探索地图\
-殖民飞船部件(太空港建成后)\\

如果提议中未包含向对方要求的内容, 就相当于一份礼物.
只有对方接受, 礼物才有效果. 与之相反的情况是, 如果
提议中未包含向对方付出的条件, 就是赤裸裸的索取.\\

从其他势力获得的先进科技, 无法直接利用, 必须先针对
该科技进行剖析性研究, 研究速度将是独自研究的两倍, 
且不必研究先期所需科技.\\

!条约\
条约分为以下几种:\\

-和平条约:
两国不得相互攻击, 也不得进入对方的国境. 
和平国家的部队对另一方的城市没有围攻效应.\\

-友好条约:
除了和平条约的上述约定之外, 
两国每回合交换各自的国家情报..\\

-盟约:
成为盟国之后, 己势力部队可在盟国国界内自由移动, 
反之亦然. 另外, 盟国间共享所有情报 (包括地区/军事
及地图情报), 对方部队的一举一动也将全部知悉.\\

另外, 没有签署任何条约的两个国家, 可以达成停战协议. 
停战协议不属于条约, 仅是一种暂时不攻击对方的允诺. 
当其中一方开战后, 协议自动失效. 
综上所述, 两国间可能的关系列表如下:\
&TREATIES\\

只有在外交交涉中, 或是一国拒绝接见
别国使节的时候, 方可取消原先签订的
条约. 取消条约后, 在接下来的三回合内
不可与同一国家再次取消其他条约.
如果A国取消与B国间的和平条约, 
则B国的同盟国也将对A国宣战.\\

!可信程度\
要判断一个国家是否言出必行, 可信度是
重要参考依据. 可信度分为以下四个等级:\
-言出必行 (100%可信度)\
-重诺守信 (90%~99%可信度)\
-捉摸不定 (70%~89%可信度)\
-言而无信 (可信度低于70%)\\

在和平时期, 每回合每个国家的可信度+1, 直至最大值. 
游戏开始时, 所有国家的最大可信度为100, 撕毁和平条约
将使当前可信度减半, 最大可信度-10. 撕毁盟约将使当前
可信度减少四分之三. 无论是否建立过外交关系, 一个国家
的可信度, 对其他国家都是公开透明的.

#MOVEMENT
$部队移动\

!部队移动\

要对部队下达命令, 请先选中执行命令的部队. 被选的部队
会以闪烁表示选中状态, 同时该部队的详细情况会显示在
屏幕下方. 用方向键, 小键盘数字键或鼠标右键指挥部队向
目标地块移动.\\

!移动限制\
通常情况下, 部队总能移动到邻接地块, 但以下情况除外:\
-目标地块上有其他势力的部队.\
-移动部队的机动力不足, 详情见下文.\
-地面部队不可在海面移动, 海上部队不可在陆地移动.\
-没有海上导航能力的舰只不可在远洋移动.\
-不可向其他势力的控制地域移动, 详情见下文.\\

!空中部队\
空中部队在每回合结束时必须回到本方基地(包括本方城市, 
军事基地, 及母舰), 如果未能返回基地, 该部队会坠毁阵亡. 
有一种方法可以避免上述情况的发生:\
::F7 给部队增加燃油\\

!机动消耗\
每次移动都需消耗一定的机动力(MP), 海上部队, 空中部队, 
以及向平坦地形移动的地面部队作斜向移动时(如西北方), 
需消耗的机动力为1点; 垂直或水平方向移动时(如正北方), 
需消耗的机动力为1.5点. 向崎岖地形移动时, 机动力为1.5的
地面部队, 所耗的机动力是平坦地形的120%, 机动力在1.5点
以上的地面部队, 所耗的机动力是平坦地形的130%. 向高山
地块移动时, 地面部队需消耗两回合机动力, 移动完成后的
一个回合将无法行动..\
::T 查看地形种类\\

移动所耗机动力的示意图如下:\
@MoveShot\
-左图 - 初始机动力1.5的部队\
-右图 - 初始机动力3的部队\\

如果部队顺着道路, 河流或运河移动, 无论原始地形为何, 
机动力消耗都将降为0.4(斜向移动)和0.6(非斜向移动).
如果顺着铁路移动, 机动力消耗将降为初始机动力的8%
(斜向移动)和12%(非斜向移动).\\
新回合开始时, 部队消耗的机动力将回复到初始最大值.
海上部队受损后, 机动力无法完全回复, 但不会低于2.5.\\
要移动到更远处, 可使用目标移动功能, 右键单击目标地块,
部队就会向该地块持续移动.\\

!控制地域\
部队分为两种: 一种有地域控制能力, 另一种则没有. 有
地域控制能力的部队在接近其他势力的地域控制部队时,
无法向对方控制的地域移动, 除非目标地块是本方城市,
或该地块存在本方的地域控制部队. 海上部队, 空中部队,
特种兵以及处于运输状态的部队没有地域控制能力.\\

!占领城市\
攻击型地面部队可占领不设防的敌方城市. 占领敌城后, 
该城隶属并在外驻扎的敌军部队将全数消亡, 整个城市
也会陷入骚乱状态, 并持续三个回合.

$战斗大全\

!攻守概论\

用方向键或鼠标可指挥本方部队攻击邻接的敌方部队, 方法
与部队移动相同. 但攻击也有一定的规则, 通常情况下, 地面
部队无法攻击海上部队, 反之亦然.  海上部队不能从城市内
或运河中发起攻击; 空中部队只要不在城市内或军事基地中, 
就只能被其他空中部队攻击. 战斗中弱势一方会被消灭, 强势
一方将受到损伤, 强弱差距越大,强势部队受损越小. 
要预知攻击的效果, 请按住Shift键, 右键点击要攻击的目标.\\

每支部队的战斗力分为攻击和防御两种. 部队情报中显示的
75/25, 意思就是攻击75, 防御25. 在一场战争中, 还有一些
因素会影响实际的战斗力:\
-部队经验: 经验上升一级, 战斗力就提升25%, 详见下文.\
-地形防御加成: 仅对被攻击一方有效, 详见地形类型一览.\
-特殊防御设施: 如城墙, 要塞等等; 仅对被攻击一方有效.\
-驻守状态: 仅对被攻击一方有效; 不处于运输状态的部队, 
若上回合没有行动, 本回合自动进入驻守状态, 防御+50%.\
-在城市里或军事基地中停留的空中部队防御力为零, 在
城市里或运河中的海上部队防御力减半.\
注意: 多种防御力加成的效果是积累相加, 而不是倍数相乘.\\

若攻击方机动不足1点, 则展开攻击时无法使出全部战斗力,
会根据剩余的机动而相应削弱.\\

!攻击成群部队\
如果两支或两支以上的部队停留在同一地块, 在遭受攻击时, 
只有一支部队出阵防守, 就是显示在地图上的那支部队. 
通常情况下, 系统会挑选防御最强的部队, 详情请点击:\\

;TILEDEF 成群部队的防守\\

如果在城市, 要塞或军事基地之外的成群部队在战斗中落败, 
该地块上所有的部队都会被消灭, 潜艇和隐形飞机也不例外.\\

!经验等级\
随着在战斗中受损状态的积累, 部队的经验等级会逐级提升,
每损失100点状态, 部队经验提升一级. 经验等级详列如下:\
&EXPERIENCE\\

!状态恢复\
回合结束时, 受损部队会根据剩余的机动相应恢复部分状态. 
如果本回合未曾移动, 部队的状态最大恢复8点, 若在未骚乱
城市或军事基地中, 状态最大恢复20点.  海面上的受损部队
状态不会恢复. 空中部队只有在城市或军事基地中才能恢复.\\

!轰炸城市\
具备轰炸能力的海上和空中部队可攻击邻接的敌方城市, 
轰炸消耗1点机动. 被轰炸的城市规模减少1级, 直至被
完全摧毁. 执行轰炸的部队不会受损, 也无法获得经验值.
轰炸方法与攻击方法相同, 用鼠标或键盘指挥部队攻击.\\

!驱逐部队\
你也可以指挥一支部队将在己方领土上驻扎的和约国部队
驱逐出境.  执行驱逐指令的部队必须是地面或海上的攻击性
部队, 且剩余机动大于1点.  驱逐的方法与攻击方法相同.

$成群部队\

!成群部队\

如果两支或两支以上的部队停留在同一地块, 在遭受攻击时, 
只有防御最强的那支部队会出阵防守, 但也会出现例外情况.
系统在确定防御部队时, 依次遵循如下规则:\
-最优先的规则是根据部队的类别. 首先是具备狂热分子属性
的地面部队, 然后是具有地域控制属性的常规地面部队, 第三
选择是没有地域控制属性的地面部队, 最后是潜艇和隐形飞机.
在运输舰和运输机上的运输部队永远不会作为防守部队出阵.\
-如果有两支部队类别相同, 防御较高的部队将担任防守任务.
这里的防御力指的是将经验等级, 当前状态和驻守状态等因素
综合考虑的结果. 对特定部队的特殊防御加成将不作考虑.\
-如果两支或两支以上具备相同的防御力(这种情况极少发生)
造价较低的那支部队将担任防守任务.

#CITIES
$城市概论\

@@43\
!城市概论\

城市的基本功能是收集邻近地块的资源, 并将收集的资源
处理后, 转为科技, 生产, 或人口. 
游戏初始时, 一个国家只有一个城市, 该城也是国家的首都. 
有两种方法增加城市数量, 一是派遣移民部队建立新城市, 
二是派兵占领别国的城市. 城市只能建立在可灌溉的土地上
(草场/平原/荒地/丘陵/苔原).\\

每个城市有各自专属的画面, 单击地图上的城市图标, 
即可打开该城市的专属画面: \\

@CityShot\

上述画面中, 主要包含了如下几个大项:\\

!市民\

城市画面左上角的人脸代表着该城的市民. 城市规模的
下限为2级, 如果规模降至2级以下, 则该城将不复存在. 
默认条件下, 城市规模上限为8级, 但上限值可通过特殊
的城市设施提升.\\

每个市民可收集一个地块(见上图1)的资源. 一个地块上
最多容纳一名市民. 被其他城市占用的地块将被标示为
不可用地块.\\

未被指定的空闲市民, 将被用作为城市戒严部队(见上图2)
可用其来维持城市安定的比例, 使城市不至于陷入骚乱.
详细信息请参照下文帮助部分.\\

!安定\

城市画面的右上方显示了当前市民的安定状态. 
该部分分为三行:\

-奢侈 (图标: %l). 如果在"经济及税收"菜单中
设定了奢侈比例, 则每个城市的贸易数中将有
一部分转化为奢侈,详见下文中的"贸易"部分.\
-安定 (%m). 默认情况下, 城市中有四名市民
是安定的, 其余市民都是不安的. 建造特定的
城市设施或奇迹, 可提升安定市民的数量. 每座
奇迹能使所在城市中的两名不安市民变为安定.\
-戒严 (%o). 每支城市戒严部队可使两名不安
市民变为安定. 另外, 特殊部队中的守城卫兵
也可使得安定市民的数量增加.\\

如果不安市民数超越了奢侈数与安定市民数
之和, 则整个城市将陷入骚乱.\\

某些特定的政体形式下, 不安事件的发生会降低
城市的安定值, 详细信息请参考\
::G 政体形式\\

!信仰\
在宗教制国家,安定值的计算与其他政体形式不同. 
每个市民都因信仰而变得安定, 因此宗教制国家
无需担心城市的安定问题, 更不会出现城市骚乱.\\


!资源种类\

城市从周围地块中收集的资源, 根据类型不同
分为三种: 粮食(%f), 原料(%p)和贸易(%t).
\\

每个地块出产资源的多少取决于以下三个条件:\
::T 地形种类\
::J 地形改造\
::G 政体形式\\

!粮食\
粮食(符号为%f)代表了各种食物. 城市粮食的生产消耗
情况显示在粮食面板中(见上图4).  粮食是人类每日的根本,
每个市民每回合需消耗2%f. 还有一些部队需要定量的粮食
作为供给, 如粮食供应不足, 将从城市的存粮中扣除. 如果
存粮耗尽, 城市陷入饥荒, 会有一名市民或一支部队消失. 
但在通常情况下, 城市生产的粮食量不会低于消耗量. 如果
生产的粮食有剩余, 多余的粮食将计入存粮, 但有一种情况
属于例外: 规模达到8的大城市每回合如果多余1单位的粮食, 
则这1单位粮食将直接转为资金, 不计入存粮. 如果存粮达到
上限, 且城市规模未达上限, 那么这个城市的规模会提升1级, 
市民数量加1.\\

!原料\
原料 (符号为%p)是一种抽象的资源, 是各种劳动生产资源
如木头, 矿石, 石油等的总称. 城市原料的生产消耗情况显示
在原料面板中(见上图5). 城市中某些隶属部队需消耗一定量
的原料以维持日常供给, 剩余原料将投入到生产建设之中. 
您可更改城市当前建设的项目(见上图6). 城市地图(见上图7)
显示了城市当前设施的位置. 
有关建设的详细信息, 请参考下一章节"生产详解".\\

!贸易\
贸易(符号为%t)代表城市的贸易所得. 只有城市周围的地块
才能进行贸易, 城市本身所在的地块不会贡献任何贸易值, 
即便城市建在金矿上也无济于事.  城市贸易的生产消耗情况
显示在贸易面板中(见上图8). 根据国家当前的政体变化, 
以及与首都的地理距离, 城市中一定比例的贸易所得会因为
腐败而白白流失, 剩余的%t将按计划中的比例分配后, 分别
用于国家税收, 奢侈消费和科学研究. 税收部分(%c)抵消支出
之后的剩余所得将充入国库; 科研部分(%r)将用作科研经费, 
加快当前科研的进度; 奢侈部分(%l)可调整市民的安定程度.  
注: 要获得贸易利润, 该城至少须拥有以下建筑:\
::B49 市政厅\\

;CORRUPTION 相关资料: 计算腐败比例\\

!资源自动分配\
指定系统对采集资源以及执行戒严的市民进行自动优化分配. 
玩家只需根据个人目标, 指定最优因子(见上图9).
自动分配有以下五种模式:
-增长最大化(默认模式). 全力保证城市人口增长, 尽一切可能
收集食物资源.\
-生产最大化. 全力保证城市设施建设, 尽可能收集原料资源. 
考虑到玩家在购买设施时需要一定财力支持, 此模式也不会
忽略了贸易资源的收集, 而粮食与科研资源将被基本忽略.\
-科研最大化. 着重收集贸易资源(即科研及税收的来源). 粮食
资源将被基本忽略.\
-生产优先. 增长及其他因素兼顾的模式, 优先考虑原料收集.\
-科研优先. 增长及其他因素兼顾的模式, 优先考虑贸易收集. \\

;OPTI 相关资料: 资源自动分配公式\\

注意, 在某些情况下, 自动分配可能无效, 例如:
在城市建设定为商品贸易时选择"生产最大化";
在城市规模发展已达上限时选择"增长最大化";
在城市无市政厅收集贸易时选择"科研最大化";
以上情况下, 在自动分配模式下的系统只能退而
求其次, 可能导致的后果难以预料.\\

有三项强制条件的优先级甚于玩家指定的最优因子, 
玩家无法改变这三项条件:\
-首要条件: 城市维持安定, 不致陷入骚乱\
-次要条件: 保证隶属部队的原料供给\
-第三条件: 保证城市人口以及隶属部队的粮食供给\\

!手动分配\
如果不想使用资源自动分配, 有两种方式切换至手动分配:
可点击圆形分配按钮底部的灰色半环; 或者直接点击上方的
城市地图. 在手动模式下, 玩家可自由分配市民在指定的地块
收集资源, 或是改为城市戒严部队. 点击在地块上工作的市民,
可将其改为城市戒严部队; 点击无人工作的地块, 可将执行
戒严的市民指派到该地块工作. 结合以上两步, 即可将一个地块
上工作的市民转移至另一地块.\\

当资源自动分配模式被关闭后, 请留心以下供求的平衡: 
粮食供给, 部队原料供给, 以及城市安定人口的比例, 否则
可能出现意料不到的后果.\\

!城市骚乱\
如果城市的不安人口数超过了安定人口数, 整个城市将陷入骚乱,
所有生产, 科研及税收活动停滞. 从骚乱爆发后的第二回合开始,
城市不再给隶属的部队供给资源, 需要资源维持的部队将逐支
解散, 每回合减少一支, 直至骚乱结束.\\

在通常情况下, 自动分配模式总能避免城市骚乱的发生, 除非
发生过多的不安事件. 如果遇到上述情况, 请将导致不安事件
的部队撤回到本土, 或解散这些部队, 即可使城市恢复秩序.\\

如果关闭了自动分配模式, 请别忘了控制城市安定人口的比例.
最常用的方法是将更多市民改为城市戒严部队.

!计算腐败比例\

一个城市的腐败比例, 根据以下公式计算:\
@CORRUPTION\
-T - 城市贸易总值\
-C - 政体腐败因子\
-D - 城市与首都的距离\
-Dmax - 理论上相隔最远的两座城市间的距离 (即以地图

如果一个国家没有首都, 则无论为何种政体, 该国所有城市

#OPTI
$资源自动分配条件公式\

!资源自动分配条件公式\

自动分配资源共包括五种情况, 其中的差别在于优先因子m的确定及计算. 优先因子m的考量范围包括以下四类: 粮食(f), 原料(p),
税收(t)和科研(r), m2为次优因子, 以此类推\
-增长最大化: m = f, m2 = 2p + t + r\
-生产最大化: m = 4p + t, m2 = r, m3 = f\
-科研最大化: m = 2r + t + p, m2 = f\
-生产优先: m = sqrt(f) * (8p + 2t + r)\
-科研优先: m = sqrt(f) * (r + t + p)


#PRODUCTION
$生产详解\

!生产目标\

建立新城或占领敌城后, 当前的建设目标将定为 "商品贸易", 
意即将生产原料直接转换为金钱利润. 点击目标图片, 可选择
其他目标, 选择范围取决于本国的科技发展水平. 按住Shift键
后点击项目, 可显示相关帮助. 有3类建设目标可供选择:\
-部队\
-城市设施\
-奇迹及国家工程\\

如果将城市当前的建设目标 更改为同类中的其他设施或部队, 
累积的生产原料将损失三分之一. 如果更改为其他类别的设施
或部队, 则累积的生产原料将直接变卖为金钱.  要想加快设施
建设的速度, 可选择已有的设施, 而后选择推倒重建, 则原设施
三分之二的生产原料将被回收, 直接添加到当前建设设施的
生产原料中; 要想加快部队建设的速度 (建设队除外), 可选择
城内驻守的部队, 而后选择整编, 则原部队三分之二的人员将
直接加入当前建设部队之中.\\

!移民\
拓荒者、工程师、奴隶以及征募部队都来自于市民, 因此在
生产这些部队后, 城市规模将会缩小. 生产征募部队和奴隶, 
规模将缩小一级; 生产拓荒者和工程师, 城市规模将缩小两级. 
移民过程是可逆的: 将以上部队加入城市, 可使城市规模扩大, 
前提是城市规模未达到上限.\\

!征募\
可以用征募的方式生产任何已知的地面部队. 征募的部队
将多耗费1%f食物. 征募的好处在于, 可将部队的生产时间
缩减到原来的三分之一. 要征募部队, 须先获得以下科技:\
::A18 征兵制度\\

!买卖生产原料\
要迅速完成生产, 可花钱购买, 购买后的目标将于下回合
开始时完工. 要想快,得花大价钱: 每个未完成的原料%p, 
需支付4倍的金钱. 反言之, 也可以变卖生产原料, 只需将
目标改为"商品贸易"即可. 累积的生产原料将转换为金钱.\\

!污染\
每1点原料生产都将造成1点污染, 而每个市民可抵消1点污染, 
多余的污染将慢慢积累为废料, 如果积累的废料数量达到240, 
城市范围内将出现一处污染区, 该地块的资源产量将降为原先
的一半. 拓荒者和工程师可以清理污染.


#BUILDINGS
$建筑介绍\

@@67\
!建筑介绍\

建筑会对所在的城市, 甚至整个国家产生积极的作用. 
共有3种建筑, 它们之间的区别在于可建造的数量:\
-城市设施: 每个城市都可建设.\
-国家工程: 每项国家工程, 同一个国家只能建设一次. 
若在新的城市建设已有的国家工程, 在新建设完工时, 
原有的国家工程将被变卖.\
-世界奇迹: 全世界只允许拥有一座. 如果某国建成了
奇迹, 则其他国家不能再建相同的奇迹. 如果建成的
奇迹随着所在城市一起被敌军摧毁, 就会永远消失, 
无法复建. 除了各自的特殊效果之外, 每座奇迹还能
使所在城市中的一名不安市民变为安定.\\

::B 城市设施\
::B201 国家工程\
::B202 世界奇迹\\

大部分城市设施和国家工程都需要现金维持, 每回合
所花费的现金将自动从国库中扣除. 如果国库耗尽, 
将随机变卖一项设施, 用以抵债.\\

在城市画面中, 按"城市设施"按钮, 可查看城市中现有的
设施列表. 如果要将某项设施变卖, 或推倒重建某设施, 
以加快当前设施的建设进度, 左键单击其图标即可. 
设施变卖的价格等同于其生产原料换算成现金的价格.


#SUPPORT
$部队隶属\

!部队隶属\

每支部队每回合需消耗1%p生产原料, 消耗原料从隶属城市
中扣除. 编制在6%w以上的船只, 需消耗双倍的生产原料. 
部队最初隶属于生产该部队的城市. 如果想改变隶属城市, 
将部队移至要隶属的城市后, 选择"改为本城部队"即可. 
城市画面中选择"隶属部队"按钮, 可查看该城的隶属部队.\\

一小部分部队不需要生产原料供给, 无需供给的部分以
灰色(%n)显示, 需要供给的部分以亮色(%p)显示. 这批
部队的数量取决于政体形式. 如果城市无法提供足够的
生产原料供给部队, 则其中最廉价的部队将被强行解散.\\

如果城市骚乱超过一个回合, 其部队供给将全部停滞, 
如果该城市存在隶属部队, 则每回合将有1支部队被解散, 
直至骚乱结束.

游戏开始画面包含了三张页面, 要新建游戏, 请在"新建游戏"

勾选"自定义玩家"框, 打开更多详细设置. 在自定义画面中, 
不仅可以对竞争种族AI, 电脑游戏难度, 人类玩家数量等等
进行详细设置, 也可展开电脑AI模块的快速对决, 或选择

勾选状态, 则这方框代表了3名玩家的共同设定. 由此可知, 
游戏玩家的上限数量为15名.\\

方框左侧的指示条代表了玩家的游戏难度, 1为简单, 3为困难. 
如果想尽可能地提高游戏难度, 可将人类玩家的难度设为最高, 
并将所有AI玩家的难度设置为最低.\\

难度越高, 人口增长, 生产速度和研究进度都会变慢, 新型兵种
的造价也会变得更为昂贵.\\

进度存档中都包含了所有的游戏回合, 因此您尽可先选择某个
更早的游戏年代, 尝试另一种策略, 也许会有不一样的结果.\\

!地图编辑\

世界尺寸和陆地比例, 再点选"编辑"运行编辑器.\\

下载地图解压后放置在游戏目录的Maps子目录下即可使用.
