&LOGO\\

!起源及目標\
C-evo is a non-commerical game project based on the famous "Sid Meier's
Civilization" by Microprose. The ambition of C-evo is not, to compete with
commercial games for the most exciting presentation. Instead, it aims at being a
true, ageless strategy game that entertains by one thing above all:
challenge. So the project focus is design of rules and AI. For more
information, visit the project homepage:\
::E2 c-evo.org\\

!所有權許可\
C-evo is a compilation of work by a number of authors. You are allowed
to play the game and to distribute it without explicit permission. The compilation
itself and all of its components (except for the graphics and the sounds) are in the Public
Domain. Means you can do with it whatever you want. This also includes the binary
code and the sources, which are available from the project homepage. As for
the graphics, you should ask the author of an item before using it for another
purpose than to play.\
::C 查看制作人員名單\\

!AI模塊\
The game has an open AI interface, which means the player can replace the standard AI
contained in the package by other AI algorithms, either for all or for single
nations. C-evo allows playing with up to 8 different AIs in the same
game. The documentation of the AI's DLL-interface is available from the project
homepage. There is also an AI development kit, you can find it in the folder where
you installed the game. Maybe competition will lead us to the first real good AI
in modern strategy gaming.\
::E1 AI模塊開發指南\\

!錯誤報告\

To support the progress of this project, please tell me about problems you have
encountered. But first you should\
-Look to the list of known bugs on the project homepage. Don't
report bugs which are already mentioned there.\
-Look to see if there is already a newer program version than yours. This might
have already fixed your problem, particularly if it's a quite conspicuous one.\
-Read the corresponding manual part. Maybe your bug isn't one. (This game is
not a Civilization II clone!)\
-Be sure it's not a problem with the installation or with some third party
expansion such as alternative AI. If you're not sure, uninstall the game with
the Windows Application Wizard, reinstall it with no expansion and then test
again. In the event of problems with an expansion, please contact its author.\\

Before reporting a problem, please try to find a way to reproduce it. A report
of a reproducable bug is much more valuable. To reproduce the bug it may well
be enough to load the book in the turn before the problem occured and continue
in the same way as before. This is even possible in case of a crash, because
there's always an autosave from the last turn, it's the book with the name
starting on "~".\\

Then, send me a detailed destription of the problem. Use the web form at\
::E3 c-evo.org/_sg/contact\
If your book is
necessary to reproduce the problem, please attach it. You can find the file in
the subfolder "Saved" of C-evo.

!快速上手\

以下是游戲基本的要點, 供新手快速入門:\

-這是一款回合制游戲, 游戲核心源自文明系列.\
-當所有部隊行動結束后, 您的回合自動結束. 如果游戲
停止運行, 也許是系統在等您手動結束回合.\
-大多數可點擊的項目都以白色外框表示.\
-在同一地塊上, 也許駐有多支部隊, 但只有最強大的部隊
才會顯示在地圖上.\
-用鼠標左鍵單擊選擇部隊, . 選中的部隊圖標將會閃爍, 
屏幕下方帶感歎號的按鈕, 是該部隊相關的操作菜單.\
-要移動部隊, 請使用鍵盤方向鍵、數字鍵或鼠標右鍵.\
-鼠標單擊城市圖標, 可顯示該城市的操作窗口, 其中最
重要的是生產按鈕, 單擊窗口中部的生產按鈕, 可選擇
新的建設目標, 例如選擇拓荒者, 去別處建立新的城市.\
-城市建設、增長的速度, 以及科研稅收的多少, 
全都取決于你的選擇.\
-在多數情況下, 在鼠標點擊某項目的同時按住Shift鍵, 
可顯示相關的幫助信息.\\\

!致文明系列的老玩家\

就地形和部隊圖片的相似程度而言, C-evo(文明進化史)
很容易被看作是文明II代經典的复制版. 但深入游戲后, 
你就會發現兩者有許多不同. 其差异程度甚至可与
文明I和II代之間的差异相提并論. 查看具体差异, 
請點擊下列標題:\
:MOVEMENT 部隊移動\
::T 地形种類\
::B202 奇跡影響\
::G 政体形式\
::A 科技演化\
:DIPLOMACY 對外交涉\
:SHIP 殖民飛船\\

除了上述差异之外, 還有以下要點提請各位玩家留意:\

-C-evo不靠隨机運算作為賣點. 每一項計划的結果都可
預期, 除了世界地圖的生成和其他勢力的舉動之外.\
-電腦計算所依据的規則与真人玩家完全相同. 電腦玩家
并無額外的优勢, 在獲得情報方面也一樣.\
-城市結构与文明系列略有出入. 在城市所在的地塊, 有1個
市民負責勞動生產, 但不會從事商業貿易; 此外, 市民結构
也有所簡化, 如去除了藝人和收稅人等等.\
-城市最小規模為2級, 生產一支拓荒者, 須耗費2個市民.\
-城市糧食儲量上限并非由城市規模決定, 因此大城市的
增長速度, 往往遠高于較小的城市.\
-無需臨近海洋或河流, 即可進行灌溉.\
-從其他勢力獲得的先進科技, 無法直接利用, 必須先針對
該科技進行剖析性研究, 研究速度將是獨自研究的兩倍, 
且不必研究先期所需科技.\
-某些高級作戰部隊無法直接建設, 必須先進行研發, 
研發成功后才能建設.\
-某些地形對地面部隊會造成傷害, 詳見"地形种類"中的"惡土".\

!歡迎進入C-evo(文明進化史)的世界\

本游戲帶您領略人類文明的發展史, 您可帶領一支种族, 
歷經六千年滄桑變遷, 從人類最初的定居開始奮斗, 求生, 
進化, 直至實現人類的終极夢想--殖民外星. 前進的道路
并不總是一番坦途: 可用的資源并不充裕, 需要精打細算; 
周圍的野地茫茫, 少說也得花几千年才能變成文明樂土; 
更別提不期而至的饑荒和變亂了, 天災人禍始終是城市
繁榮發展的最大威脅..\\

种种不利因素之下, 唯有靠深謀遠慮才能帶領人民走向幸福.
擴張帝國領土, 發展城市建設, 修兵營, 辦大學, 造工厂......
在建設的同時, 也別忽視了科學研究, 研發新式部隊和武器, 
派遣拓荒部隊開辟新的沃土, 引領船隊尋找海洋彼岸的大陸, 
將整個世界盡收眼底.\\

別忘了, 你的國家并不是這世界中唯一的文明. 每塊大陸上
都有不同的部落, 存在相當的文化差异. 要記住, 沒有哪個
种族是天命永存的, 任何一個帝國都有可能在一夜間傾覆, 
你的國家也不例外.\\

与他國的第一次接触也許并不會是一次愉快的經歷, 但通過
不懈的外交交涉, 許多原先的不可能也會變成可能. 与其他
文明交換科技知識, 用條約保護各自的利益, 或是建立堅固
的盟約, 在陷入麻煩時有個幫手應該不是坏事.  但也別太過
依賴條約, 因為這個世界沒有永遠的朋友, 只有生存的對手, 
不論是哪個种族, 都在為了同一個目標而競爭: 最終的胜利.


#BASICS
$概論\

!概論\

國家的組成單位是部隊和城市. 部隊的職責是探險和攻守, 
而城市的職能是人口增長, 生產建設和科學研究.\\

世界地圖由四邊形地塊組成. 每個地塊都有其地形屬性, 
根据不同的地形种類, 地塊的以下三种屬性會有所不同: 
資源產量, 防御加成, 和移動消耗. 同一地塊上最多能建
一座城市, 但可容納的同國部隊數量并無限制. 要獲得
特定地塊的幫助信息, 請在點選地塊時按住Shift鍵.\
::T 地形种類\\

本游戲采用了回合制進行方式，其他對手必須等待您全部
行動完畢，結束回合之后方可依次行動. 在默認配置下, 
所有部隊行動完成后,回合將自動結束, 如果您看到"等待
回合結束"的提示,請點擊屏幕右下角的結束回合按鈕.\\

#DIPLOMACY
$外交\

!概論\

兩個素未謀面的國家要建立外交關系, 必須等到其中一國
的地面部隊發現另一國的地面部隊或城市之后方可進行. 
在國家情報畫面, 可查看已知其他勢力的相關信息. 
在國家情報畫面中, 顯示了一個國家的政体形式, 現有資金, 
外交立場, 以及人口, 領土, 科技, 探索等概況. \\

!交涉\
當兩國發生接触之后, 可開展外交交涉, 在交涉中可進行
交換貿易或簽署協約. 整個交涉過程主要由制定提議, 
接受提議, 以及取消條約三部分組成.\\

當交涉雙方都停止提議的時候, 交涉就將結束. 如果一方
選擇終止談判, 交涉也會立即終止.\\

!提議\
外交交涉主要的組成部分是各种提議. 提議的內容中包含了
交涉一方向另一方提出的要求, 以及愿意為之付出的條件. 
可能包含的內容有:\
-一項條約\
-國家情報\
-軍事情報\
-一筆現金\
-科技成果\
-部隊研發\
-探索地圖\
-殖民飛船部件(太空港建成后)\\

如果提議中未包含向對方要求的內容, 就相當于一份禮物.
只有對方接受, 禮物才有效果. 与之相反的情況是, 如果
提議中未包含向對方付出的條件, 就是赤裸裸的索取.\\

從其他勢力獲得的先進科技, 無法直接利用, 必須先針對
該科技進行剖析性研究, 研究速度將是獨自研究的兩倍, 
且不必研究先期所需科技.\\

!條約\
條約分為以下几种:\\

-和平條約:
兩國不得相互攻擊, 也不得進入對方的國境. 
和平國家的部隊對另一方的城市沒有圍攻效應.\\

-友好條約:
除了和平條約的上述約定之外, 
兩國每回合交換各自的國家情報..\\

-盟約:
成為盟國之后, 己勢力部隊可在盟國國界內自由移動, 
反之亦然. 另外, 盟國間共享所有情報 (包括地區/軍事
及地圖情報), 對方部隊的一舉一動也將全部知悉.\\

另外, 沒有簽署任何條約的兩個國家, 可以達成停戰協議. 
停戰協議不屬于條約, 僅是一种暫時不攻擊對方的允諾. 
當其中一方開戰后, 協議自動失效. 
綜上所述, 兩國間可能的關系列表如下:\
&TREATIES\\

只有在外交交涉中, 或是一國拒絕接見
別國使節的時候, 方可取消原先簽訂的
條約. 取消條約后, 在接下來的三回合內
不可与同一國家再次取消其他條約.
如果A國取消与B國間的和平條約, 
則B國的同盟國也將對A國宣戰.\\

!可信程度\
要判斷一個國家是否言出必行, 可信度是
重要參考依据. 可信度分為以下四個等級:\
-言出必行 (100%可信度)\
-重諾守信 (90%~99%可信度)\
-捉摸不定 (70%~89%可信度)\
-言而無信 (可信度低于70%)\\

在和平時期, 每回合每個國家的可信度+1, 直至最大值. 
游戲開始時, 所有國家的最大可信度為100, 撕毀和平條約
將使當前可信度減半, 最大可信度-10. 撕毀盟約將使當前
可信度減少四分之三. 無論是否建立過外交關系, 一個國家
的可信度, 對其他國家都是公開透明的.

#MOVEMENT
$部隊移動\

!部隊移動\

要對部隊下達命令, 請先選中執行命令的部隊. 被選的部隊
會以閃爍表示選中狀態, 同時該部隊的詳細情況會顯示在
屏幕下方. 用方向鍵, 小鍵盤數字鍵或鼠標右鍵指揮部隊向
目標地塊移動.\\

!移動限制\
通常情況下, 部隊總能移動到鄰接地塊, 但以下情況除外:\
-目標地塊上有其他勢力的部隊.\
-移動部隊的机動力不足, 詳情見下文.\
-地面部隊不可在海面移動, 海上部隊不可在陸地移動.\
-沒有海上導航能力的艦只不可在遠洋移動.\
-不可向其他勢力的控制地域移動, 詳情見下文.\\

!空中部隊\
空中部隊在每回合結束時必須回到本方基地(包括本方城市, 
軍事基地, 及母艦), 如果未能返回基地, 該部隊會墜毀陣亡. 
有一种方法可以避免上述情況的發生:\
::F7 給部隊增加燃油\\

!机動消耗\
每次移動都需消耗一定的机動力(MP), 海上部隊, 空中部隊, 
以及向平坦地形移動的地面部隊作斜向移動時(如西北方), 
需消耗的机動力為1點; 垂直或水平方向移動時(如正北方), 
需消耗的机動力為1.5點. 向崎嶇地形移動時, 机動力為1.5的
地面部隊, 所耗的机動力是平坦地形的120%, 机動力在1.5點
以上的地面部隊, 所耗的机動力是平坦地形的130%. 向高山
地塊移動時, 地面部隊需消耗兩回合机動力, 移動完成后的
一個回合將無法行動..\
::T 查看地形种類\\

移動所耗机動力的示意圖如下:\
@MoveShot\
-左圖 - 初始机動力1.5的部隊\
-右圖 - 初始机動力3的部隊\\

如果部隊順著道路, 河流或運河移動, 無論原始地形為何, 
机動力消耗都將降為0.4(斜向移動)和0.6(非斜向移動).
如果順著鐵路移動, 机動力消耗將降為初始机動力的8%
(斜向移動)和12%(非斜向移動).\\
新回合開始時, 部隊消耗的机動力將回复到初始最大值.
海上部隊受損后, 机動力無法完全回复, 但不會低于2.5.\\
要移動到更遠處, 可使用目標移動功能, 右鍵單擊目標地塊,
部隊就會向該地塊持續移動.\\

!控制地域\
部隊分為兩种: 一种有地域控制能力, 另一种則沒有. 有
地域控制能力的部隊在接近其他勢力的地域控制部隊時,
無法向對方控制的地域移動, 除非目標地塊是本方城市,
或該地塊存在本方的地域控制部隊. 海上部隊, 空中部隊,
特种兵以及處于運輸狀態的部隊沒有地域控制能力.\\

!占領城市\
攻擊型地面部隊可占領不設防的敵方城市. 占領敵城后, 
該城び屬并在外駐扎的敵軍部隊將全數消亡, 整個城市
也會陷入騷亂狀態, 并持續三個回合.

!部隊指令\

部隊种類不同, 以及部隊所處地形不同, 部隊指令也有差异:\

-解散: 解散該部隊\
-整編: 將部隊編制打散, 加入到當前城市建設的新部隊中\
-目標移動: 繼續向先前設定的目標移動\
-稍后行動: 暫時跳過, 選擇下一部隊\
-原地等待: 本回合不再行動\
-駐守警戒: 持續不行動, 直至被再次手動選中\
-恢复休整: 受損部隊在原地停留恢复狀態, 直至完全康复\
-改為本城部隊: 將所在城市指定為該部隊新的び屬城市\
-返回最近城市: 盡快返回距离最近的本方城市\
-界面居中: 以該部隊為窗口中心, 調整游戲窗口\
-掠奪: 摧毀當前地塊的地形建設(需耗費1點机動力)

#COMBAT
$戰斗大全\

!攻守概論\

用方向鍵或鼠標可指揮本方部隊攻擊鄰接的敵方部隊, 方法
与部隊移動相同. 但攻擊也有一定的規則, 通常情況下, 地面
部隊無法攻擊海上部隊, 反之亦然.  海上部隊不能從城市內
或運河中發起攻擊; 空中部隊只要不在城市內或軍事基地中, 
就只能被其他空中部隊攻擊. 戰斗中弱勢一方會被消滅, 強勢
一方將受到損傷, 強弱差距越大,強勢部隊受損越小. 
要預知攻擊的效果, 請按住Shift鍵, 右鍵點擊要攻擊的目標.\\

每支部隊的戰斗力分為攻擊和防御兩种. 部隊情報中顯示的
75/25, 意思就是攻擊75, 防御25. 在一場戰爭中, 還有一些
因素會影響實際的戰斗力:\
-部隊經驗: 經驗上升一級, 戰斗力就提升25%, 詳見下文.\
-地形防御加成: 僅對被攻擊一方有效, 詳見地形類型一覽.\
-特殊防御設施: 如城牆, 要塞等等; 僅對被攻擊一方有效.\
-駐守狀態: 僅對被攻擊一方有效; 不處于運輸狀態的部隊, 
若上回合沒有行動, 本回合自動進入駐守狀態, 防御+50%.\
-在城市里或軍事基地中停留的空中部隊防御力為零, 在
城市里或運河中的海上部隊防御力減半.\
注意: 多种防御力加成的效果是積累相加, 而不是倍數相乘.\\

若攻擊方机動不足1點, 則展開攻擊時無法使出全部戰斗力,
會根据剩余的机動而相應削弱.\\

!攻擊成群部隊\
如果兩支或兩支以上的部隊停留在同一地塊, 在遭受攻擊時, 
只有一支部隊出陣防守, 就是顯示在地圖上的那支部隊. 
通常情況下, 系統會挑選防御最強的部隊, 詳情請點擊:\\

;TILEDEF 成群部隊的防守\\

如果在城市, 要塞或軍事基地之外的成群部隊在戰斗中落敗, 
該地塊上所有的部隊都會被消滅, 潛艇和隱形飛机也不例外.\\

!經驗等級\
隨著在戰斗中受損狀態的積累, 部隊的經驗等級會逐級提升,
每損失100點狀態, 部隊經驗提升一級. 經驗等級詳列如下:\
&EXPERIENCE\\

!狀態恢复\
回合結束時, 受損部隊會根据剩余的机動相應恢复部分狀態. 
如果本回合未曾移動, 部隊的狀態最大恢复8點, 若在未騷亂
城市或軍事基地中, 狀態最大恢复20點.  海面上的受損部隊
狀態不會恢复. 空中部隊只有在城市或軍事基地中才能恢复.\\

!轟炸城市\
具備轟炸能力的海上和空中部隊可攻擊鄰接的敵方城市, 
轟炸消耗1點机動. 被轟炸的城市規模減少1級, 直至被
完全摧毀. 執行轟炸的部隊不會受損, 也無法獲得經驗值.
轟炸方法与攻擊方法相同, 用鼠標或鍵盤指揮部隊攻擊.\\

!驅逐部隊\
你也可以指揮一支部隊將在己方領土上駐扎的和約國部隊
驅逐出境.  執行驅逐指令的部隊必須是地面或海上的攻擊性
部隊, 且剩余机動大于1點.  驅逐的方法与攻擊方法相同.


#TILEDEF
$成群部隊\

!成群部隊\

如果兩支或兩支以上的部隊停留在同一地塊, 在遭受攻擊時, 
只有防御最強的那支部隊會出陣防守, 但也會出現例外情況.
系統在确定防御部隊時, 依次遵循如下規則:\
-最优先的規則是根据部隊的類別. 首先是具備狂熱分子屬性
的地面部隊, 然后是具有地域控制屬性的常規地面部隊, 第三
選擇是沒有地域控制屬性的地面部隊, 最后是潛艇和隱形飛机.
在運輸艦和運輸机上的運輸部隊永遠不會作為防守部隊出陣.\
-如果有兩支部隊類別相同, 防御較高的部隊將擔任防守任務.
這里的防御力指的是將經驗等級, 當前狀態和駐守狀態等因素
綜合考慮的結果. 對特定部隊的特殊防御加成將不作考慮.\
-如果兩支或兩支以上具備相同的防御力(這种情況极少發生)
造价較低的那支部隊將擔任防守任務.

#CITIES
$城市概論\

@@43\
!城市概論\

城市的基本功能是收集鄰近地塊的資源, 并將收集的資源
處理后, 轉為科技, 生產, 或人口. 
游戲初始時, 一個國家只有一個城市, 該城也是國家的首都. 
有兩种方法增加城市數量, 一是派遣移民部隊建立新城市, 
二是派兵占領別國的城市. 城市只能建立在可灌溉的土地上
(草場/平原/荒地/丘陵/苔原).\\

每個城市有各自專屬的畫面, 單擊地圖上的城市圖標, 
即可打開該城市的專屬畫面: \\

@CityShot\

上述畫面中, 主要包含了如下几個大項:\\

!市民\

城市畫面左上角的人臉代表著該城的市民. 城市規模的
下限為2級, 如果規模降至2級以下, 則該城將不复存在. 
默認條件下, 城市規模上限為8級, 但上限值可通過特殊
的城市設施提升.\\

每個市民可收集一個地塊(見上圖1)的資源. 一個地塊上
最多容納一名市民. 被其他城市占用的地塊將被標示為
不可用地塊.\\

未被指定的空閒市民, 將被用作為城市戒嚴部隊(見上圖2)
可用其來維持城市安定的比例, 使城市不至于陷入騷亂.
詳細信息請參照下文幫助部分.\\

!安定\

城市畫面的右上方顯示了當前市民的安定狀態. 
該部分分為三行:\

-奢侈 (圖標: %l). 如果在"經濟及稅收"菜單中
設定了奢侈比例, 則每個城市的貿易數中將有
一部分轉化為奢侈,詳見下文中的"貿易"部分.\
-安定 (%m). 默認情況下, 城市中有四名市民
是安定的, 其余市民都是不安的. 建造特定的
城市設施或奇跡, 可提升安定市民的數量. 每座
奇跡能使所在城市中的兩名不安市民變為安定.\
-戒嚴 (%o). 每支城市戒嚴部隊可使兩名不安
市民變為安定. 另外, 特殊部隊中的守城衛兵
也可使得安定市民的數量增加.\\

如果不安市民數超越了奢侈數与安定市民數
之和, 則整個城市將陷入騷亂.\\

某些特定的政体形式下, 不安事件的發生會降低
城市的安定值, 詳細信息請參考\
::G 政体形式\\

!信仰\
在宗教制國家,安定值的計算与其他政体形式不同. 
每個市民都因信仰而變得安定, 因此宗教制國家
無需擔心城市的安定問題, 更不會出現城市騷亂.\\


!資源种類\

城市從周圍地塊中收集的資源, 根据類型不同
分為三种: 糧食(%f), 原料(%p)和貿易(%t).
\\

每個地塊出產資源的多少取決于以下三個條件:\
::T 地形种類\
::J 地形改造\
::G 政体形式\\

!糧食\
糧食(符號為%f)代表了各种食物. 城市糧食的生產消耗
情況顯示在糧食面板中(見上圖4).  糧食是人類每日的根本,
每個市民每回合需消耗2%f. 還有一些部隊需要定量的糧食
作為供給, 如糧食供應不足, 將從城市的存糧中扣除. 如果
存糧耗盡, 城市陷入饑荒, 會有一名市民或一支部隊消失. 
但在通常情況下, 城市生產的糧食量不會低于消耗量. 如果
生產的糧食有剩余, 多余的糧食將計入存糧, 但有一种情況
屬于例外: 規模達到8的大城市每回合如果多余1單位的糧食, 
則這1單位糧食將直接轉為資金, 不計入存糧. 如果存糧達到
上限, 且城市規模未達上限, 那么這個城市的規模會提升1級, 
市民數量加1.\\

!原料\
原料 (符號為%p)是一种抽象的資源, 是各种勞動生產資源
如木頭, 礦石, 石油等的總稱. 城市原料的生產消耗情況顯示
在原料面板中(見上圖5). 城市中某些び屬部隊需消耗一定量
的原料以維持日常供給, 剩余原料將投入到生產建設之中. 
您可更改城市當前建設的項目(見上圖6). 城市地圖(見上圖7)
顯示了城市當前設施的位置. 
有關建設的詳細信息, 請參考下一章節"生產詳解".\\

!貿易\
貿易(符號為%t)代表城市的貿易所得. 只有城市周圍的地塊
才能進行貿易, 城市本身所在的地塊不會貢獻任何貿易值, 
即便城市建在金礦上也無濟于事.  城市貿易的生產消耗情況
顯示在貿易面板中(見上圖8). 根据國家當前的政体變化, 
以及与首都的地理距离, 城市中一定比例的貿易所得會因為
腐敗而白白流失, 剩余的%t將按計划中的比例分配后, 分別
用于國家稅收, 奢侈消費和科學研究. 稅收部分(%c)抵消支出
之后的剩余所得將充入國庫; 科研部分(%r)將用作科研經費, 
加快當前科研的進度; 奢侈部分(%l)可調整市民的安定程度.  
注: 要獲得貿易利潤, 該城至少須擁有以下建筑:\
::B49 市政廳\\

;CORRUPTION 相關資料: 計算腐敗比例\\

!資源自動分配\
指定系統對采集資源以及執行戒嚴的市民進行自動优化分配. 
玩家只需根据個人目標, 指定最优因子(見上圖9).
自動分配有以下五种模式:
-增長最大化(默認模式). 全力保證城市人口增長, 盡一切可能
收集食物資源.\
-生產最大化. 全力保證城市設施建設, 盡可能收集原料資源. 
考慮到玩家在購買設施時需要一定財力支持, 此模式也不會
忽略了貿易資源的收集, 而糧食与科研資源將被基本忽略.\
-科研最大化. 著重收集貿易資源(即科研及稅收的來源). 糧食
資源將被基本忽略.\
-生產优先. 增長及其他因素兼顧的模式, 优先考慮原料收集.\
-科研优先. 增長及其他因素兼顧的模式, 优先考慮貿易收集. \\

;OPTI 相關資料: 資源自動分配公式\\

注意, 在某些情況下, 自動分配可能無效, 例如:
在城市建設定為商品貿易時選擇"生產最大化";
在城市規模發展已達上限時選擇"增長最大化";
在城市無市政廳收集貿易時選擇"科研最大化";
以上情況下, 在自動分配模式下的系統只能退而
求其次, 可能導致的后果難以預料.\\

有三項強制條件的优先級甚于玩家指定的最优因子, 
玩家無法改變這三項條件:\
-首要條件: 城市維持安定, 不致陷入騷亂\
-次要條件: 保證び屬部隊的原料供給\
-第三條件: 保證城市人口以及び屬部隊的糧食供給\\

!手動分配\
如果不想使用資源自動分配, 有兩种方式切換至手動分配:
可點擊圓形分配按鈕底部的灰色半環; 或者直接點擊上方的
城市地圖. 在手動模式下, 玩家可自由分配市民在指定的地塊
收集資源, 或是改為城市戒嚴部隊. 點擊在地塊上工作的市民,
可將其改為城市戒嚴部隊; 點擊無人工作的地塊, 可將執行
戒嚴的市民指派到該地塊工作. 結合以上兩步, 即可將一個地塊
上工作的市民轉移至另一地塊.\\

當資源自動分配模式被關閉后, 請留心以下供求的平衡: 
糧食供給, 部隊原料供給, 以及城市安定人口的比例, 否則
可能出現意料不到的后果.\\

!城市騷亂\
如果城市的不安人口數超過了安定人口數, 整個城市將陷入騷亂,
所有生產, 科研及稅收活動停滯. 從騷亂爆發后的第二回合開始,
城市不再給び屬的部隊供給資源, 需要資源維持的部隊將逐支
解散, 每回合減少一支, 直至騷亂結束.\\

在通常情況下, 自動分配模式總能避免城市騷亂的發生, 除非
發生過多的不安事件. 如果遇到上述情況, 請將導致不安事件
的部隊撤回到本土, 或解散這些部隊, 即可使城市恢复秩序.\\

如果關閉了自動分配模式, 請別忘了控制城市安定人口的比例.
最常用的方法是將更多市民改為城市戒嚴部隊.

!計算腐敗比例\

一個城市的腐敗比例, 根据以下公式計算:\
@CORRUPTION\
-T - 城市貿易總值\
-C - 政体腐敗因子\
-D - 城市与首都的距离\
-Dmax - 理論上相隔最遠的兩座城市間的距离 (即以地圖
全高与地圖半寬為直角邊, 所得的三角形斜邊長度)\\

如果一個國家沒有首都, 則無論為何种政体, 該國所有城市
中的腐敗比例為100%, 有法院的城市腐敗比例為50%.


#OPTI
$資源自動分配條件公式\

!資源自動分配條件公式\

自動分配資源共包括五种情況, 其中的差別在于优先因子m的确定及計算. 优先因子m的考量范圍包括以下四類: 糧食(f), 原料(p),
稅收(t)和科研(r), m2為次优因子, 以此類推\
-增長最大化: m = f, m2 = 2p + t + r\
-生產最大化: m = 4p + t, m2 = r, m3 = f\
-科研最大化: m = 2r + t + p, m2 = f\
-生產优先: m = sqrt(f) * (8p + 2t + r)\
-科研优先: m = sqrt(f) * (r + t + p)

$生產詳解\

!生產目標\

建立新城或占領敵城后, 當前的建設目標將定為 "商品貿易", 
意即將生產原料直接轉換為金錢利潤. 點擊目標圖片, 可選擇
其他目標, 選擇范圍取決于本國的科技發展水平. 按住Shift鍵
后點擊項目, 可顯示相關幫助. 有3類建設目標可供選擇:\
-部隊\
-城市設施\
-奇跡及國家工程\\

如果將城市當前的建設目標 更改為同類中的其他設施或部隊, 
累積的生產原料將損失三分之一. 如果更改為其他類別的設施
或部隊, 則累積的生產原料將直接變賣為金錢.  要想加快設施
建設的速度, 可選擇已有的設施, 而后選擇推倒重建, 則原設施
三分之二的生產原料將被回收, 直接添加到當前建設設施的
生產原料中; 要想加快部隊建設的速度 (建設隊除外), 可選擇
城內駐守的部隊, 而后選擇整編, 則原部隊三分之二的人員將
直接加入當前建設部隊之中.\\

!移民\
拓荒者、工程師、奴び以及征募部隊都來自于市民, 因此在
生產這些部隊后, 城市規模將會縮小. 生產征募部隊和奴び, 
規模將縮小一級; 生產拓荒者和工程師, 城市規模將縮小兩級. 
移民過程是可逆的: 將以上部隊加入城市, 可使城市規模擴大, 
前提是城市規模未達到上限.\\

!征募\
可以用征募的方式生產任何已知的地面部隊. 征募的部隊
將多耗費1%f食物. 征募的好處在于, 可將部隊的生產時間
縮減到原來的三分之一. 要征募部隊, 須先獲得以下科技:\
::A18 征兵制度\\

!買賣生產原料\
要迅速完成生產, 可花錢購買, 購買后的目標將于下回合
開始時完工. 要想快,得花大价錢: 每個未完成的原料%p, 
需支付4倍的金錢. 反言之, 也可以變賣生產原料, 只需將
目標改為"商品貿易"即可. 累積的生產原料將轉換為金錢.\\

!污染\
每1點原料生產都將造成1點污染, 而每個市民可抵消1點污染, 
多余的污染將慢慢積累為廢料, 如果積累的廢料數量達到240, 
城市范圍內將出現一處污染區, 該地塊的資源產量將降為原先
的一半. 拓荒者和工程師可以清理污染.

$建筑介紹\

@@67\
!建筑介紹\

建筑會對所在的城市, 甚至整個國家產生積极的作用. 
共有3种建筑, 它們之間的區別在于可建造的數量:\
-城市設施: 每個城市都可建設.\
-國家工程: 每項國家工程, 同一個國家只能建設一次. 
若在新的城市建設已有的國家工程, 在新建設完工時, 
原有的國家工程將被變賣.\
-世界奇跡: 全世界只允許擁有一座. 如果某國建成了
奇跡, 則其他國家不能再建相同的奇跡. 如果建成的
奇跡隨著所在城市一起被敵軍摧毀, 就會永遠消失, 
無法复建. 除了各自的特殊效果之外, 每座奇跡還能
使所在城市中的一名不安市民變為安定.\\

::B 城市設施\
::B201 國家工程\
::B202 世界奇跡\\

大部分城市設施和國家工程都需要現金維持, 每回合
所花費的現金將自動從國庫中扣除. 如果國庫耗盡, 
將隨机變賣一項設施, 用以抵債.\\

在城市畫面中, 按"城市設施"按鈕, 可查看城市中現有的
設施列表. 如果要將某項設施變賣, 或推倒重建某設施, 
以加快當前設施的建設進度, 左鍵單擊其圖標即可. 
設施變賣的价格等同于其生產原料換算成現金的价格.


#SUPPORT
$部隊び屬\

!部隊び屬\

每支部隊每回合需消耗1%p生產原料, 消耗原料從び屬城市
中扣除. 編制在6%w以上的船只, 需消耗雙倍的生產原料. 
部隊最初び屬于生產該部隊的城市. 如果想改變び屬城市, 
將部隊移至要び屬的城市后, 選擇"改為本城部隊"即可. 
城市畫面中選擇"び屬部隊"按鈕, 可查看該城的び屬部隊.\\

一小部分部隊不需要生產原料供給, 無需供給的部分以
灰色(%n)顯示, 需要供給的部分以亮色(%p)顯示. 這批
部隊的數量取決于政体形式. 如果城市無法提供足夠的
生產原料供給部隊, 則其中最廉价的部隊將被強行解散.\\

如果城市騷亂超過一個回合, 其部隊供給將全部停滯, 
如果該城市存在び屬部隊, 則每回合將有1支部隊被解散, 
直至騷亂結束.
